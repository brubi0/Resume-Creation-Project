# Resume Rules

These are the standards every resume must meet. Apply these during Phase 3 (Resume Transformation).

---

## The 6 Evaluation Criteria

### 1. Clarity
- Structure is logical and easy to follow
- Section headers are clear
- Job titles, companies, and dates are unambiguous
- No confusing jargon or acronyms without context

### 2. Focus
- Resume is targeted to a **single primary role**
- No irrelevant roles or filler content
- Every bullet supports the candidate's positioning for that specific target
- If the candidate targets multiple roles, produce separate resume versions or pick a primary angle and note which bullets to swap for alternates

### 3. Quantification
- Every bullet has a number: $, %, time, volume, accuracy, scale
- Numbers should be **bolded** for skimability
- If you cannot quantify it, **cut it** — no exceptions

### 4. Results-Driven / ROI Impact
- Bullets show outcomes, not just duties
- Format: "Did X, resulting in Y"
- Include dollar impact where possible
- Show business value, not just tasks completed

### 5. Cuts Through the Noise
- Top third of resume must hook the reader in 7-10 seconds
- Lead with credentials/differentiators
- Key achievements visible immediately
- Strong professional summary with metrics

### 6. Skimability
- Can be scanned in 7-10 seconds
- Bold key metrics
- Consistent formatting
- Digestible bullet length (1-2 lines max)

## Resume Structure (Mandatory Order)

This order is not a suggestion. Follow it exactly. The goal is to front-load proof so a recruiter sees value before they see lists.

### 1. Header
- Name + Title/Credential on the same line (e.g., "JANE SMITH | Senior NetSuite Administrator (8x Certified)")
- The title must match the primary target role — this is the first thing a recruiter reads
- Contact info (phone, email, LinkedIn)
- Languages / Work authorization (if relevant)

### 2. Professional Summary (3-4 lines)
- Lead with differentiator (certifications, years, specialty)
- Include 3-4 key metrics inline
- End with scope: users managed, systems implemented, team size, etc.
- **Must be targeted to the primary role** — not a generic catch-all
- The summary should answer: "Who is this person, what scale do they operate at, and what's their biggest win?"

**Example:**
> 8x NetSuite-certified Administrator with 10+ years delivering ERP implementations, WMS deployments, and system optimization across manufacturing, distribution, and medical device operations. Career highlights: 4 successful ERP migrations, 3 WMS go-lives, 90%+ reduction in picking errors, and 30% faster system performance. Trained 100+ users and built 30+ SOPs to drive adoption and audit readiness.

### 3. Key Achievements (4-6 bullets) — MANDATORY
This section is **required**. It is the hook that makes a recruiter want to keep reading. Without it, the recruiter has to dig through role-by-role bullets to figure out who the candidate is.

- Career-level themes, not job-specific details
- Each bullet is a category with proof points
- Numbers required on every line
- Should answer: "If I only read 4 lines, what would I know about this person?"

**Example format:**
- **ERP Implementation:** Led NetSuite migration from QuickBooks/Fishbowl with zero major post-go-live issues for a $7M distribution company
- **Cost Impact:** Delivered $100K+ in one-time savings through vendor analysis and process optimization
- **Compliance & Risk:** Identified 4 missing state tax registrations across 2 employers, avoiding $125K+ in penalties

### 4. Professional Experience
- **Current/Recent roles:** 4-6 bullets each, results-focused
- **Older roles:** 2-4 bullets each
- **Pre-2015 or 10+ years ago:** Collapse to 1-2 lines or "Earlier Experience" section
- Each role should open with a 1-line context sentence: company size, revenue, team, what the situation was

### 5. Certifications
- List on one line if possible
- Most relevant first
- In-progress certifications are fine but label them clearly

### 6. Core Competencies
- Group by category (4-5 categories max)
- 4-8 skills per category
- Senior-level skills only
- This section goes LAST because it's a reference list, not proof

### 7. Education
- Include only if it strengthens the application
- **Has a degree:** List it with school name, degree, field. Dates optional for experienced candidates.
- **Partial coursework / no degree:** Omit the section entirely if 10+ years of experience and certifications compensate. Let the career speak.
- **Degree in unrelated field:** Include but don't highlight — just the one-liner.
- **Never list "courses" without a degree** — it signals "no degree" louder than just omitting the section

## Formatting Rules

- **Bold** all key metrics
- Remove roles older than 10 years or collapse to "Earlier Experience"
- Remove junior-level skills from Core Competencies
- Eliminate repetition — say it once, strongly. If the same achievement appears in two roles, keep it in the more impressive one and cut it from the other
- Every bullet must have a number or be cut
- Frame weak percentages positively (2% reduction → 98% availability)
- Start every bullet with a strong action verb
- Keep bullets to 1-2 lines max
- **Never use weak verbs:** "started", "assisted", "helped", "participated in", "was responsible for", "was involved in". Replace with what you actually did.
- **Certifications should NOT be in a table** — tables can break ATS parsing. Use a single line or short list.

## Weak Bullet Filter

Before presenting the draft, scan every bullet for these red flags. Any bullet that triggers a flag must be rewritten or cut:

| Red Flag | Example | Fix |
|----------|---------|-----|
| No number | "Designed landed cost procedures" | Add the outcome: "...reducing margin calculation errors by X%" or cut |
| Starts with "Started" / "Assisted" / "Helped" | "Started implementing fixed asset management" | What was the result? If none, cut it |
| Task-only (no outcome) | "Configured roles, permissions, and reports" | Add scope + impact: "Configured X roles and Y reports for Z clients, reducing support tickets by N%" |
| Repeated across roles | Same achievement in two jobs | Keep in the stronger role, cut from the other |
| Passive voice | "Was responsible for reconciliations" | "Reduced reconciliation time from X to Y days" |
| Generic / could apply to anyone | "Managed financial operations" | What specifically? What scale? What changed? |

## Multiple Target Roles

When a candidate lists multiple target roles:

1. **Pick a primary angle** — Ask which role they'd accept tomorrow if offered. That's the primary.
2. **Build the resume for the primary** — Summary, Key Achievements, and bullet emphasis all target this role
3. **Note alternates** — In the discovery file, document which bullets to swap or re-emphasize for alternate targets
4. **Offer to generate variants** — If the roles are significantly different (e.g., Controller vs. Consultant), recommend separate resume versions with different summaries and Key Achievements sections

A resume that tries to target 5 roles targets none of them.

## Resume Length

| Experience Level | Target Length |
|------------------|---------------|
| Entry-level (< 5 years) | 1 page |
| Mid-level (5-15 years) | 2 pages |
| Senior/Executive (15+ years) | 2 pages (3 max if justified) |

## Bullet Transformation Examples

### Before (Task-Focused)
> Responsible for bank reconciliations and financial reporting

### After (Results-Focused)
> Reduced month-end close from **12 days to 5 days** through automated bank reconciliation workflows and streamlined financial reporting

---

### Before (No Numbers)
> Managed integrations between NetSuite and external platforms

### After (Quantified)
> Managed **6** platform integrations (Avalara, Stripe, Shopify, Celigo, Pacejet, Envoy B2B) processing **500-1,000** transactions monthly with **<1%** sync failure rate

---

### Before (Weak Percentages)
> Reduced stockouts by 2% and assembly delays by 3%

### After (Positive Framing)
> Configured assembly workflows achieving **98%** inventory availability and **97%** on-time assembly completion during post-COVID supply chain disruptions

---

### Before (Weak Verb)
> Started implementing fixed asset management processes

### After (Outcome-Focused)
> Implemented fixed asset tracking for **$2M** in company assets, establishing depreciation schedules and reducing year-end audit prep by **40 hours**

*If you don't have the outcome data, cut the bullet entirely.*

## The 7-Second Test

A hiring manager scanning for 7 seconds should immediately see:
1. Who you are (title + credential **in the header**, not buried in the summary)
2. Your scale (years, company sizes, user counts)
3. Your top 3-4 wins (in the Key Achievements section)
4. Why you are different from 50 other applicants

If any of these are buried or missing, the resume fails the test.

---

## ATS Compatibility Rules

Many resumes are rejected by Applicant Tracking Systems before a human sees them:

- **No tables** for critical content (certifications, skills). Use simple lists or inline text
- **No columns or text boxes** — ATS may not parse them correctly
- **Use standard section headers:** "Professional Experience", "Education", "Certifications", "Core Competencies". Creative headers like "Where I've Made Impact" confuse ATS parsers
- **Include exact keywords from the job posting** — If the posting says "month-end close," use "month-end close," not "period-end procedures"
- **Simple formatting only** — Bold is fine. Avoid graphics, icons, shading, or complex layouts

## Final Checklist

Run this checklist against the draft BEFORE presenting it. Every item must pass.

- [ ] Header includes name + target role title (not just name)
- [ ] Summary has 3+ metrics inline and is targeted to the primary role
- [ ] **Key Achievements section exists** with 4-6 quantified career-level bullets
- [ ] Every experience bullet has a number ($/%, time, volume, scale)
- [ ] No bullets with weak verbs (started, assisted, helped, participated)
- [ ] No bullets without an outcome (task-only bullets are cut)
- [ ] All key numbers are **bolded**
- [ ] No repetition of same achievement across sections
- [ ] No junior-level skills in Core Competencies
- [ ] Old roles are minimized or removed
- [ ] Section order is: Header → Summary → Key Achievements → Experience → Certifications → Core Competencies
- [ ] Certifications are NOT in a table
- [ ] Education is handled appropriately (omitted if no degree and 10+ years experience)
- [ ] Resume passes the 7-second test
- [ ] Length is appropriate for experience level
- [ ] No ATS-hostile formatting (tables for content, columns, text boxes)
