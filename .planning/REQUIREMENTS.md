# Requirements: Resume Creation Project

**Defined:** 2026-03-23
**Core Value:** Every resume produced must make a recruiter stop and call the candidate

## v1 Requirements

Requirements for current milestone. Validated items are already shipped.

### Workflow Engine

- [x] **WORK-01**: System reads workflow.md as entry point and drives all phases automatically
- [x] **WORK-02**: System detects experience level and routes to appropriate track
- [x] **WORK-03**: Discovery answers saved incrementally to candidate discovery.md
- [x] **WORK-04**: Interrupted sessions can be resumed from where they left off
- [x] **WORK-05**: System can update an existing resume without re-doing full interview

### Discovery Process

- [x] **DISC-01**: Experienced track covers 12 question areas (target role, narrative, differentiator, achievements, metrics, leadership, skills emphasis, gaps, certs, education, context, skills matrix)
- [x] **DISC-02**: Early-career track covers 9 question areas (direction, education, projects, work experience, internships, certs/skills, activities, differentiator, skills matrix)
- [x] **DISC-03**: Questions asked one at a time with push-back on vague answers
- [x] **DISC-04**: Probing techniques documented for both tracks

### Resume Quality

- [x] **QUAL-01**: Resume follows mandatory section order per track
- [x] **QUAL-02**: Every bullet passes weak bullet filter (no numberless, no weak verbs, no task-only)
- [x] **QUAL-03**: Self-audit runs against full checklist before presenting draft
- [x] **QUAL-04**: Recruiter eye test simulates 7-second scan and reports impression
- [x] **QUAL-05**: ATS compatibility rules enforced (no tables for content, standard headers)
- [x] **QUAL-06**: Multi-target role strategy with primary angle and variant support

### Industry Profiles

- [x] **PROF-01**: Profiles define skill categories, proficiency levels, and interview questions per role
- [x] **PROF-02**: NetSuite Administrator profile exists as reference implementation
- [x] **PROF-03**: New profiles auto-generated by researching 5-10 current job postings via web search
- [ ] **PROF-04**: Existing profiles can be cloned and tweaked for related roles

### Deliverables

- [x] **DELV-01**: Resume generated as markdown, converted to DOCX via pandoc with template
- [x] **DELV-02**: Interview prep guide with STAR-format answers for 5-7 strongest bullets
- [x] **DELV-03**: Skills matrix as HTML with color-coded proficiency levels
- [x] **DELV-04**: Score card grading resume against 6 evaluation criteria
- [ ] **DELV-05**: Cover letter targeted to specific job postings (optional deliverable)
- [ ] **DELV-06**: LinkedIn profile optimization guide (optional deliverable)

### Candidate Management

- [x] **CAND-01**: Per-candidate directory structure (input, discovery, output)
- [x] **CAND-02**: Candidate data gitignored for privacy
- [x] **CAND-03**: generate.sh resolves candidate output paths automatically
- [ ] **CAND-04**: HTML dashboard showing all candidates, status, scores, deliverables

### Job Targeting

- [ ] **TARG-01**: System can ingest a specific job posting URL or text
- [ ] **TARG-02**: Keywords extracted from job posting and woven into resume
- [ ] **TARG-03**: Resume variant generated targeting the specific posting

## v2 Requirements

Deferred to future milestone.

### Advanced Features

- **ADV-01**: Resume version history with diff tracking
- **ADV-02**: Batch processing for multiple candidates
- **ADV-03**: Resume comparison tool (before vs. after scoring)
- **ADV-04**: Template library with multiple DOCX styles per industry

## Out of Scope

| Feature | Reason |
|---------|--------|
| Web application / SaaS | This is a local Claude Code workflow, not a hosted product |
| Resume parsing / OCR | Candidates provide their own files |
| Job board integration | Creates resumes, not applications |
| Payment / billing | Free tool for personal use |
| AI resume screening | We create resumes, not evaluate incoming ones |
